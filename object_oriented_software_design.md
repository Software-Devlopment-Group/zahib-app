## Object-Oriented Software Design

The Zahib Food Ordering System is designed using object-oriented programming principles to ensure modularity, maintainability, and scalability.

The system is divided into several classes, each representing a real-world entity in the application:

- User Class: Handles customer registration and login functions.
- Order Class: Manages order placement, tracking, and rating.
- FoodItem Class: Represents menu items with their names and prices.
- Staff Class: Allows staff members to view orders and update order status.
- Admin Class: Manages menu updates and report generation.

### Design Principles Applied:

1. Encapsulation:
Each class contains its own attributes and methods, keeping related data together.

2. Abstraction:
Only essential functions are exposed to users, hiding internal complexity.

3. Modularity:
Each class performs a separate responsibility, making the system easier to maintain.

4. Reusability:
Classes can be reused and extended for future improvements.

This object-oriented design improves code organization and makes the system scalable for future development.
